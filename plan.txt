1 项目定位与核心目标

1.1 问题背景

在图像分类任务中，训练集往往被当成“既定事实”：只要丢给模型训练即可。但现实中，数据集普遍存在这些问题：

含有噪声标注、低质量样本；

类内冗余严重，大量“长得差不多”的样本反复出现；

决策边界附近的关键样本比例不足；

难以评估后续新采集数据的价值。

传统数据选择／数据裁剪方法（如基于 loss、EL2N、GraNd、Forgetting、MoSo 等）有两个共性缺陷：

主要针对“已有训练集”，难以直接对未见样本打分；

强烈依赖特定训练过程和模型结构，缺乏可复用、可解释的“样本质量模型”。


1.2 本项目的目标

本项目要做的不是简单的“再造一个数据选择算法”，而是设计一个面向图像分类任务的样本质量评估框架，具有以下能力：

(1) 给每个样本一个可解释的质量得分，明确区分“好样本”“一般样本”“坏样本”：

得分越高，表示语义更干净；

在类空间中覆盖更充分；

在类内关键变化方向上更有价值。

(2) 这个得分不仅能作用于当前训练集，还能对训练阶段未见的新样本进行快速评分：

新采集的数据不必重新训练就能知道“值不值得加入训练集”。

(3) 得分可以自然用于数据选择／数据裁剪：

例如指定只保留 top 50% 的高质量样本；

或在样本总量固定的约束下，用 Selection Optimization 选出最优子集。

(4) 权重学习：

静态指标本身是 train-free 的（基于 CLIP 特征和类内结构），但各指标的重要性会随数据集、任务而变化；

我们使用一次代理模型训练动态（earlyloss、coverage、stability）来学习这些静态指标的权重，得到一个“数据驱动”的评分模型；

训练动态只参与“教学”，不直接参与最终评分，保证最终评分函数能泛化到未见样本。

(5) 在静态指标之前引入轻量的 Dataset Adapter，对 CLIP 图像特征与类别文本特征做数据集级的联合适配，使得静态指标在当前数据集上更稳定、更合理，同时不破坏对未见样本的泛化能力。

2 整体框架概览

从流程视角，整个框架可以分为四个阶段：

2.1 特征抽取与 Dataset Adapter 训练

使用预训练 CLIP 的图像编码器和文本编码器；

冻结 CLIP 主体参数，在图像端与文本端分别训练一个轻量、维度保持的线性 Dataset Adapter，并使用英文类别 prompt 进行图文对齐训练，使当前数据集上的图像特征与类别语义更加一致；

后续静态评分统一基于 adapter 后的共同嵌入空间进行：图像侧得到适配后的图像特征 g_i，文本侧得到适配后的类别语义原型 t̃_c。

2.2 静态多指标评分（Static Scoring）

基于 adapter 后的特征 g_i，构造三个 train-free 静态指标：

SA：语义对齐度（Semantic Alignment）；

Div：类内多样性覆盖度（基于同类 kNN 距离均值的稀疏度）；

DDS：类内难度方向得分（基于 PCA 的低方差方向投影）。

2.3 从训练动态学习权重（Weight Learning via Dynamics）

在原始图像上以 K 折交叉验证训练一个代理分类模型（如 ResNet-18），按折记录训练子集与验证子集（OOF）的 logits 轨迹；

基于训练轨迹构造训练视角分量 A/B/C/R，刻画“吸收效率、边界推进信息、覆盖互补增益、极端噪声风险”；

基于 OOF 验证轨迹构造验证视角分量 T/V，分别刻画“可迁移增益”和“后期持久难度”；

将 A/B/C/T/V/R 融合并归一化得到动态效用标签 u_i；

用带 L2 正则的线性回归（Ridge），将 u_i 回归到 (SA, Div, DDS)，学习出静态指标的权重 w。

2.4 基于最终得分进行数据选择与 Selection Optimization

最终评分定义为 Score(x) = Σ_k ẇ_k · feature_k，其中 ẇ 由 w 归一化而来；

直接用 Score 排序可以做简单数据选择；

进一步，在 Selection Optimization 中引入可学习的选择变量 d_i，在给定选样比例约束下，优化“整体被选子集的总 Score”。

2.5 整体实现效果

train-free 静态评分（可泛化到未见样本）；

利用一次训练动态学习权重（提高评分模型与任务的一致性）；

支持静态排序选择和基于优化的 Selection Optimization；

在静态指标前加入 Dataset Adapter，提升 CLIP 特征在目标数据集上的适配性。

3 Dataset Adapter 模块设计

3.1 设计目标

引入 Dataset Adapter 的目的：

适配预训练 CLIP 在目标数据集（如 CIFAR-10/100、Tiny-ImageNet）上的分布偏移，使类内结构与类间语义更清晰；
提升 SA、Div、DDS 这些静态指标在当前数据集上的表达质量；
仅在特征层做轻量变换，不改变 CLIP 主干参数；
不破坏“静态指标 → 动态监督 → 权重学习 → 静态评分函数”的整体逻辑；
对未见样本依然可直接前向计算 adapter 并评分。

与仅适配图像侧相比，同时引入图像端与文本端 adapter 的优势在于：类别语义原型本身也会受到提示词形式与域偏移的影响；在冻结 CLIP 的前提下，让两端都通过小参数模块吸收“数据集特有知识”，可以更稳定地改善图文对齐质量，从而为后续的 margin-式 SA 与结构性指标（Div / DDS）提供更可靠的共同嵌入空间。

3.2 结构形式（双端、维度保持的单层线性 Adapter）

设 E_I(·) 为冻结的 CLIP 图像编码器，E_T(·) 为冻结的 CLIP 文本编码器。对第 i 个图像样本 I_i：

f_i = E_I(I_i) ∈ ℝ^d,     f_i ← L2Norm(f_i)

对类别 c 的文本提示词 T_c（prompt）：

T_c = "a photo of a [CLS]"，其中 [CLS] 用类别名称替换；对 Tiny-ImageNet，类别名称不是目录中的 WNID，而是通过 wnids.txt / words.txt 映射得到的英文类别名；
u_c = E_T(T_c) ∈ ℝ^d,     u_c ← L2Norm(u_c)

为进行数据集层面的适配，我们为两种模态分别引入“维度保持（dimension-preserving）”的线性 adapter：

图像端 adapter：
g_i = A_I(f_i) = W_I f_i + b_I

文本端 adapter：
t̃_c = A_T(u_c) = W_T u_c + b_T

其中：

W_I ∈ ℝ^{d×d},   W_T ∈ ℝ^{d×d}；
两端都只做单层线性映射，不设置中间瓶颈层，不使用额外非线性激活；
L2 归一化不写在 adapter 模块内部，而是在后续训练相似度计算与静态指标计算时显式进行。

该设计具有：

参数量远小于 CLIP 主体，仅做“轻量迁移”而非重训练；
在保持接口简洁的前提下，与当前项目代码中的 YangCLIP 风格 adapter 实现一致；
同时修正图像分布偏移与文本原型偏移，提升图文对齐的一致性；
适配后的共同空间更利于 PCA / kNN 等结构性操作（Div / DDS）。

3.3 训练目标与流程（冻结 CLIP，仅训练双端 adapter）

训练目标采用 batch 内双向 InfoNCE / softmax-contrastive 范式，使同一 batch 中的图像表示与其对应类别文本表示在适配后空间中更接近，并与 batch 内其他配对拉开距离。对一个 batch 中的第 i 个样本，先取其图像特征 f_i 以及对应标签 y_i 的类别文本特征 u_{y_i}，再经 adapter 得到：

g_i = A_I(f_i),     t̃_i = A_T(u_{y_i})

在计算相似度前，对两侧表示分别做 L2 归一化。设 batch 大小为 B，定义相似度矩阵：

S_{ij} = cos(g_i, t̃_j) / τ

其中 cos(·,·) 为余弦相似度，τ 为温度系数（默认 0.07）。

损失函数采用双向交叉熵形式的 InfoNCE：

L_image = - (1/B) Σ_{i=1}^B log ( exp(S_{ii}) / Σ_{j=1}^B exp(S_{ij}) )

L_text = - (1/B) Σ_{i=1}^B log ( exp(S_{ii}) / Σ_{j=1}^B exp(S_{ji}) )

L_adapter = 0.5 · (L_image + L_text)

这等价于在 batch 内同时约束“图像检索文本”和“文本检索图像”两个方向，更符合当前代码中的默认训练方式。

训练细节与默认参数（按当前项目代码实现）：

优化器：Adam；
初始学习率：1e-4；
训练轮数：30 epochs；
学习率调度：StepLR(step_size=30, gamma=0.1)；
batch size：默认对 CIFAR-10 / CIFAR-100 / Tiny-ImageNet 统一取 256（也允许通过命令行参数覆盖）；
温度系数：τ=0.07（默认固定）。

训练过程：

(1) 冻结 CLIP 的所有参数，仅训练 A_I 与 A_T 的参数（W_I, b_I, W_T, b_T）；
(2) 在完整训练集上，先用固定 prompt 为所有类别预编码文本特征 u_c；训练时对每个 batch，根据样本标签从这些类别文本特征中取出对应的 batch 文本特征，再与 batch 图像特征一起送入双端 adapter；
(3) 对 adapter 输出显式做 L2 归一化，基于 batch 内双向 InfoNCE 损失更新双端 adapter；
(4) 训练完成后固定 A_I 与 A_T，后续所有静态指标与评分计算都基于 (g_i, t̃_c)，不再更新 adapter。

3.4 与后续模块的关系

顺序关系：

先训练双端 Adapter（A_I, A_T）并固定；
再基于适配后的共同空间计算 SA、Div、DDS；
再在原始图像上以 K 折交叉验证训练代理模型并记录训练动态（该动态构造不依赖 adapter）；
最后用动态构造 u_i，并用 (SA, Div, DDS) 回归 u_i 学习静态权重 ẇ。

因此不存在“动态影响 adapter，再用动态监督静态指标”的循环依赖，逻辑清晰稳定。

对未见样本：

对新样本 x_new，先用 CLIP 提取 f_new，再经图像端 adapter 得到 g_new；
并使用固定的类别文本原型 t̃_c（由英文 prompt → E_T → A_T 得到；Tiny-ImageNet 仍先做 WNID 到英文类别名的映射）；
在实际计算 SA / Div / DDS 时，再对所需特征显式做 L2 归一化；
最后用固定 ẇ 得到 Score(x_new)。


4 静态多指标样本评分模块（基于 adapter 后特征）

这一部分完全不依赖代理模型训练动态，是支撑“未见样本评分”的关键；也是后续动态权重学习的输入特征来源。

记：

训练样本 (x_i, y_i)；

CLIP 图像编码器输出 f_i，图像端 Adapter 输出 g_i；
CLIP 文本编码器对类别提示词输出 u_c，文本端 Adapter 输出适配后的类别原型 t̃_c。

其中，adapter 本身只做线性映射；在后续 SA、Div、DDS 的实际计算中，再按需要对 g_i 与 t̃_c 显式做 L2 归一化。

所有静态指标均在 g_i 所在的特征空间中构建。

4.1 语义对齐度（Semantic Alignment, SA）

语义对齐度（Semantic Alignment, SA）用于度量样本图像内容与其标注类别语义之间的一致性。直观地说，如果一张图像确实属于类别 y_i，那么它在高维语义空间中应当更接近“该类别的语义描述”，而远离其他类别的语义描述。本项目基于冻结的 CLIP 图像编码器和文本编码器，在此基础上引入 Dataset Adapter，对图像侧特征进行数据集层面的轻量适配，并在适配后的特征空间中定义 SA 作为样本质量的一个核心指标。

本节首先给出基于余弦相似度的原始 SA 定义，然后在此基础上引入 margin 形式的改进版 SA，解释其设计动机、数学形式及实现方案。

4.1.1 特征表示与基础设定

设 E_I(·) 为冻结的 CLIP 图像编码器，E_T(·) 为冻结的 CLIP 文本编码器。为适配目标数据集分布，我们在图像侧与文本侧同时引入 Dataset Adapter（A_I, A_T），得到适配后的多模态表示：

g_i = A_I(E_I(I_i)),

其中 I_i 为第 i 个图像样本，g_i ∈ R^d 为该样本在适配后语义空间中的图像表示。若未启用图像端 Adapter，则有 g_i = E_I(I_i)。

对于类别 c，对其构造文本描述 prompt，例如：

T_c = "a photo of a [CLS]"（用类别名称替换 [CLS]）；在 Tiny-ImageNet 上，[CLS] 对应的是由 wnids.txt / words.txt 解析得到的英文类别名，而不是目录 WNID。

先通过文本编码器获得基础语义嵌入：

u_c = E_T(T_c),

再经文本端 Adapter 得到适配后的类别语义原型：

t̃_c = A_T(u_c).

其中 u_c, t̃_c ∈ R^d。在实际计算余弦相似度、margin 以及基于内积的距离时，再对 g_i 与 t̃_c 显式做 L2 归一化，以保证几何可比性。

4.1.2 原始 SA 定义：正类语义相似度（Cosine-based SA）

在最基础的设计中，语义对齐度 SA_i 定义为样本图像与其标注类别文本特征之间的余弦相似度：

SA_i^(cos) = cos(g_i, t̃_{y_i}),

其中 y_i 为样本 i 的标注类别。该定义反映了样本在语义空间中与“正确类别”之间的接近程度，具有以下特征：

1）形式简单、可解释性强：
SA_i^(cos) 越大，说明图像在语义空间上越接近其标注类别的文本描述，直观表示“图像语义与标签是否匹配”。

2）实现方便：
在实际实现中，只需对 g_i 和 t̃_{y_i} 做 L2 归一化，然后做一次点积即可得到 SA_i^(cos)。

然而，SA_i^(cos) 仅关注“图像与正类的相似度”，并未显式考虑“图像与其他类别的区分度”。在实际分类任务中，如果一个样本同时与多个类别都具有较高相似度，那么即使它对正类相似度较高，这个样本仍然是“语义模糊”或“容易混淆”的样本，属于质量可疑的样本类型。为增强 SA 对这种情况的敏感度，本项目在上述基础上进一步引入 margin 形式的语义对齐度。

4.1.3 改进的 margin 语义对齐度（Margin-based SA）

为了同时刻画“对正类的贴合程度”和“与其他类别的区分程度”，本项目将 SA 定义为正类相似度与最相似负类相似度之间的差值，即 margin：

SA_i = cos(g_i, t̃_{y_i}) − max_{c ≠ y_i} cos(g_i, t̃_c).

其中：

1）cos(g_i, t̃_{y_i})：
表示样本在语义空间中与标注类别语义的贴合程度，越大越好。

2）max_{c ≠ y_i} cos(g_i, t̃_c)：
表示样本在“所有非标注类别”中，与其最相似的那个负类的语义相似度。如果这个值很高，说明该样本对其他类别也容易产生混淆。

因此，SA_i 的含义可以解释为：

SA_i 大：样本不仅与正类高度对齐，而且与所有负类保持足够间隔，语义清晰、标签可靠、视觉信息明确；

SA_i 小甚至为负：样本与正类的相似度有限，或与某个负类几乎一样接近，说明样本存在语义模糊、标注有问题、信息不足或噪声较重等情况。

与原始的 SA_i^(cos) 相比，margin 形式的 SA 具备以下优势：

1）更符合分类任务的判别逻辑：
分类问题的本质不仅是“与正确类别接近”，更是“与正确类别相比，和其他类别拉开足够差距”。margin 形式正是把这种“相对优势”显式量化出来。

2）对噪声样本和模糊样本更加敏感：
噪声样本往往在多个类别上都有较高相似度，此时 cos(g_i, t̃_{y_i}) 可能仍然不低，但 max_{c ≠ y_i} cos(g_i, t̃_c) 也会较高，导致 margin 下降。这样就能有效将“语义模糊样本”标记为低质量样本。

3）与 Adapter 训练目标保持一致：
Adapter 的训练过程本身就是通过对比学习提高正类对齐、削弱负类混淆。用 margin SA 作为质量指标，可以更自然地反映 Adapter 对“语义空间可分性”的改善。

4）可解释性更强：
对于某个样本，既可以查看 cos(g_i, t̃_{y_i}) 看是否“正类相似度不足”，也可以查看 max_{c ≠ y_i} cos(g_i, t̃_c) 看是否是“被某个负类强烈吸引”，从而给出更细粒度的分析。

4.1.4 SA 的数学形式与实现步骤

在具体实现中，考虑一批样本的图像特征和全部类别的文本特征，定义：

G = [g_1; g_2; …; g_N] ∈ R^{N×d},
T = [t_1; t_2; …; t_C] ∈ R^{C×d},

其中每一行是一个 L2 归一化后的特征向量。通过矩阵乘法可一次性得到所有样本在所有类别上的余弦相似度：

S = G T^T ∈ R^{N×C}.

对于第 i 个样本，其标注类别为 y_i，则有：

1）正类相似度：
s_i^{(pos)} = S[i, y_i] = cos(g_i, t̃_{y_i}).

2）最相似负类相似度：
先在第 i 行中屏蔽掉列 y_i，对剩余列求最大值：
s_i^{(neg)} = max_{c ≠ y_i} S[i, c].

3）margin 式语义对齐度：
SA_i = s_i^{(pos)} − s_i^{(neg)}.

此过程可以通过简单而高效的张量操作实现：

使用矩阵乘法得到 S；

通过 gather 操作取出正类相似度；

构造布尔掩码屏蔽正类位置，在其余位置上求行最大值作为负类相似度；

二者相减得到 SA_i。

在代码中，SA 计算函数 _margin_similarity 即实现上述逻辑，支持批量计算并兼容不同设备（CPU/GPU）。

4.1.5 原始 SA 与 margin SA 在项目中的关系

本项目在设计上同时保留“原始 SA（仅正类余弦）”和“margin SA（正类−最相似负类）”两种视角：

1）原始 SA（cosine-based）：
SA_i^(cos) = cos(g_i, t̃_{y_i})
主要用于直观评估“样本与其标签语义的正向契合程度”，在一些分析中可以直接使用这一简单指标，便于解释和展示。

2）margin SA（margin-based）：
SA_i = cos(g_i, t̃_{y_i}) − max_{c ≠ y_i} cos(g_i, t̃_c)
作为本项目实际使用的主版本 SA，用于样本质量评分、数据选择、以及后续与 SDS、DDS 等指标联合分析。它在反映样本“是否容易被其他类别混淆”方面具有更强的能力，更符合样本质量评估任务的需求。

在后续实验与分析中，本项目会以 margin SA 为主，必要时对比展示原始 SA^(cos)，以便分离“正类对齐问题”和“负类混淆问题”。

4.1.6 归一化（更新：类内 0.2% 分位点 min-max 归一化，支持未见样本外推）

由于 SA 将与 Div、DDS 线性加权得到总分 Score，并进一步用于排序选择与未见样本质量评估，因此必须将 SA 映射到统一尺度 [0,1]。与传统的全局 min-max 不同，本项目采用“类内分位点 min-max”作为固定标尺：用训练集每个类别内部的 0.2% 与 99.8% 分位点作为有效范围端点，并对超界样本进行饱和截断，从而兼顾鲁棒性与可外推性。

记 SA 的原始 margin 分值为 SA_raw(i)，样本 i 的类别为 y_i=c。对每个类别 c，在训练集内收集该类所有 SA_raw，计算：

q_low^c = Quantile(SA_raw | y=c, 0.002)
q_high^c = Quantile(SA_raw | y=c, 0.998)

对属于类别 c 的任意样本 i（包括训练集样本与未见样本），定义类内归一化后的 SA 分数为：

SA_i = clip( (SA_raw(i) - q_low^c) / (q_high^c - q_low^c), 0, 1 )

其中 clip(·,0,1) 表示将结果截断到 [0,1] 区间；这意味着：

若 SA_raw(i) ≤ q_low^c，则 SA_i = 0（极低语义对齐样本）；

若 SA_raw(i) ≥ q_high^c，则 SA_i = 1（极高语义对齐样本）；

若 SA_raw(i) 处于 [q_low^c, q_high^c] 内，则线性映射到 (0,1) 区间，保留“幅度差异”。

数值退化处理：若某类出现 q_high^c ≈ q_low^c（该类 SA_raw 几乎为常数），则该类所有样本 SA_i 统一置为 0.5。

该归一化方式的优点在于：
1）尺度对齐：与 Div、DDS 同一 [0,1] 空间，便于线性加权；
2）鲁棒性：不受极端异常值影响（分位点而非 min/max）；
3）可外推：未见样本即使超出训练集范围，也会自然饱和到 0 或 1，不会产生不可控的 >1/<0。

4.2 多样性覆盖度 Div（基于类内前 k 近邻距离均值的稀疏度）

4.2.1 设计动机

在图像分类任务中，单个样本的价值不仅取决于其语义是否清晰、是否与类别语义对齐，还取决于它在所属类别内部所处的位置。若一个样本周围已经存在大量与其高度相似的同类样本，则该样本对类别覆盖范围的扩展作用较弱；相反，若一个样本位于类别内部相对稀疏的区域，则它更可能提供额外的局部结构信息，有助于提高训练子集对类内分布的覆盖能力。

基于这一考虑，本研究将样本在类内特征空间中的“局部稀疏程度”作为多样性覆盖度的基础刻画。Div 指标的目标不是衡量样本是否“难”，也不是衡量样本是否“正确”，而是衡量该样本是否处于能够补充类别内部覆盖范围的位置。因此，Div 更强调样本在类内分布中的相对空间作用。

4.2.2 基本表示

设训练集为
D = {(x_i, y_i)}_{i=1}^N，
其中 y_i ∈ {1, 2, ..., C} 为类别标签。

首先，利用 CLIP 图像编码器提取样本的视觉特征，并通过训练得到的图像侧 adapter 进行映射，得到样本 x_i 的特征表示
z_i ∈ R^d。

对每一个类别 c，记其样本集合为
D_c = {x_i | y_i = c}，
对应特征集合为
Z_c = {z_i | y_i = c}。

Div 的计算完全在类内进行，即仅在同类别样本之间比较相对位置，而不直接引入跨类别距离。

4.2.3 类内前 k 近邻距离均值

对于类别 c 中任一样本 x_i，设其特征为 z_i。记同类其他样本构成的候选邻域为
N_c(i) = {z_j | y_j = c, j ≠ i}。

在该集合中，按欧氏距离从小到大排序，取前 k 个最近邻。这里的 k 既可以直接给定整数，也可以给定一个比例参数。当 k 取小数且满足 0 < k < 1 时，将其解释为该类别样本数的比例，并转换为实际邻居个数：
k_c = ceil(k · |D_c|)。

同时，为保证可计算性，实际使用时将 k_c 截断到区间
[1, |D_c| - 1]
内，即至少取 1 个邻居，至多不超过该类除自身外的样本数。

在此基础上，定义样本 x_i 的原始多样性距离为其类内前 k_c 个最近邻欧氏距离的平均值：
d_i^{Div} = (1 / k_c) ∑_{z_j ∈ KNN_c(i)} ||z_i - z_j||_2，

其中 KNN_c(i) 表示样本 x_i 在类别 c 内的前 k_c 个最近邻集合。

该量的含义是：若样本周围存在大量与其高度相似的同类样本，则平均近邻距离较小，说明该样本位于类别内部较为密集、冗余度较高的区域；若平均近邻距离较大，则说明该样本位于类别内部相对稀疏的位置，具有更强的覆盖补充作用。

4.2.4 类内分位点归一化

不同类别在特征尺度、类内紧致程度、样本数量等方面可能存在差异，因此原始距离 d_i^{Div} 不能直接跨类别比较。为消除类别间尺度差异，本研究在每个类别内部对原始距离进行分位点归一化。

对于类别 c，收集该类所有样本的原始距离集合
{d_i^{Div} | y_i = c}。

记该集合的低分位点与高分位点分别为
q_c^{low} = Quantile({d_i^{Div}}, 0.002),
q_c^{high} = Quantile({d_i^{Div}}, 0.998)。

则样本 x_i 的最终 Div 分数定义为
Div_i = clip((d_i^{Div} - q_c^{low}) / (q_c^{high} - q_c^{low}), 0, 1), 当 y_i = c。

其中 clip(·, 0, 1) 表示将结果截断到 [0, 1] 区间内。

这种处理方式的作用是：

保留类内样本稀疏程度的相对排序关系；
减弱极端异常值对归一化区间的影响；
使不同类别上的 Div 分数具有可比较的统一量纲。

若某一类别中所有样本的原始距离几乎相同，即
q_c^{low} ≈ q_c^{high}，
则说明该类别在当前特征空间中的类内距离区分度很弱。此时，为避免数值不稳定，统一将该类别样本的 Div 分数置为 0.5，表示其中性、多样性信息不足以进一步区分样本优劣。

4.2.5 指标含义与作用

按照上述定义，Div_i 越大，表示样本 x_i 在所属类别内部越处于相对稀疏的位置，越可能对类内覆盖范围的扩展具有正向作用；Div_i 越小，则表示样本更靠近局部密集区域，其信息更可能已被其他同类样本充分覆盖。

因此，Div 的核心作用不是偏向“代表性中心样本”，而是鼓励保留那些能够在类别内部补充局部结构信息、降低冗余度的样本。它所刻画的是类内分布覆盖层面的样本价值，与 SA 所强调的语义一致性、与 DDS 所强调的边界与困难方向信息形成互补。

4.2.6 实现说明

特征空间：
Div 使用 CLIP 图像编码器提取特征，并通过训练后的图像侧 adapter 进行映射后计算类内距离。这样做的目的，是在保留预训练视觉语义结构的基础上，使特征表示更贴合当前数据集的类别分布。
距离计算范围：
Div 严格在同类别内部计算近邻距离，不将跨类别样本纳入近邻集合。这样可以避免类别间语义差异过大对类内覆盖估计造成干扰。
默认参数：
默认取 k = 0.05，即对每个类别使用该类别约 5% 的样本数作为近邻数量，并通过向上取整与边界截断得到实际的 k_c。
数值处理：
实际实现中先基于归一化特征的内积关系计算平方欧氏距离，再恢复为欧氏距离并求前 k 个最近邻的均值，从而提高计算效率并保持与理论定义一致。

4.2.7 小结

Div 指标从“类内局部稀疏性”的角度刻画样本质量，将样本是否能够补充类别覆盖范围作为评价重点。其基本思想是：在同类别特征空间中，越位于稀疏区域的样本，越可能提供额外的分布信息；越位于密集区域的样本，越可能具有较强冗余性。通过类内前 k 近邻距离均值与分位点归一化，Div 为后续静态综合评分提供了一个与语义一致性、困难方向信息互补的多样性分量。

4.3 类内难度方向得分 DDS（基于类内 PCA 低方差方向投影的困难性刻画）

4.3.1 设计动机

在图像分类任务中，样本的重要性并不只体现在其是否处于类别中心区域，也不只体现在其是否能够补充类内覆盖范围。对于许多类别而言，一部分样本虽然数量较少、偏离主流分布，但它们可能恰好对应类别内部较少被观察到、却对分类边界有重要作用的变化方向。若训练子集完全由大量“典型样本”构成，则模型虽能较好学习类别主体结构，却可能缺乏对这些少见但关键变化模式的感知能力。

基于这一考虑，本研究引入类内难度方向得分 DDS（Difficulty Direction Score），用于从类别内部特征分布中识别那些沿着“低方差但仍具有结构意义的方向”发生较大偏移的样本。DDS 的核心思想是：对于某一类别，主方差方向通常描述该类别内部最常见、最稳定的变化模式，而低方差方向则更可能承载少见、细粒度、非主流但具有判别价值的局部变化；若某个样本在这些方向上的偏移较大，则说明它可能包含更强的边界信息或困难信息，从而具有更高的样本价值。

4.3.2 基本表示

设训练集为
D = {(x_i, y_i)}_{i=1}^N，
其中 y_i ∈ {1, 2, ..., C} 为类别标签。

首先，利用 CLIP 图像编码器提取样本的视觉特征，并通过训练得到的图像侧 adapter 进行映射，得到样本 x_i 的特征表示
z_i ∈ R^d。

对每一个类别 c，记其样本集合为
D_c = {x_i | y_i = c}，
对应特征集合为
Z_c = {z_i | y_i = c}。

DDS 的全部计算均在类内进行，即每个类别独立建立自己的局部统计结构，而不直接引入跨类别样本。

4.3.3 类内 PCA 建模

对于类别 c，对其特征集合 Z_c 求均值向量
μ_c = (1 / |D_c|) ∑_{y_i=c} z_i。

定义中心化特征为
\tilde{z}_i = z_i - μ_c。

基于该类别的中心化特征，构造协方差矩阵
Σ_c = (1 / (|D_c| - 1)) ∑_{y_i=c} \tilde{z}_i \tilde{z}_i^T。

为提高数值稳定性，实际实现中在协方差矩阵上加入一个很小的对角正则项，即
Σ_c^{reg} = Σ_c + λI，
其中默认
λ = 1e-6。

随后对 Σ_c^{reg} 做特征分解，得到特征值与对应特征向量：
Σ_c^{reg} u_j = λ_j u_j，
其中
λ_1 ≤ λ_2 ≤ ... ≤ λ_d。

这里的特征值按从小到大排序，因此前面的方向对应类别内部方差较小的变化方向。

4.3.4 困难方向集合的选择规则

DDS 不直接使用全部 PCA 方向，而是从低方差方向中选择一个“困难方向集合” U_c。该集合的选择同时受三个因素控制：

最少方向数参数 k；
特征值累计占比下界 eigval_lower_bound；
特征值累计占比上界 eigval_upper_bound。

默认参数为：
k = 5，
eigval_lower_bound = 0.02，
eigval_upper_bound = 0.2。

设该类别所有特征值之和为
S_c = ∑{j=1}^d λ_j，
并定义从小到大累计的能量占比：
r_t = (∑{j=1}^t λ_j) / S_c。

困难方向的选择步骤如下。

（1）确定起始位置

若 eigval_lower_bound > 0，则先寻找所有满足
r_t < eigval_lower_bound
的位置。若存在这样的 t，则将起始位置设置为其中最后一个位置的下一个方向；也就是说，先跳过累计能量仍明显过小的一段最小特征值方向。

若不存在任何满足
r_t < eigval_lower_bound
的位置，说明从最小特征值开始累计时，其占比已经不低于下界。此时，若特征维度大于 1，则直接将起始位置设为第 2 个方向，即跳过最小特征值方向；若维度只有 1，则起始位置仍为第 1 个方向。

当 eigval_lower_bound = 0 时，起始位置直接设为第 1 个方向。

（2）在上界约束下向后累积选择

从起始位置开始，按特征值从小到大的顺序依次尝试加入方向。若加入当前方向后，已选方向对应特征值之和仍不超过
eigval_upper_bound · S_c，
则保留该方向；一旦超过上界，则停止继续加入。

因此，最终选择的方向集合 U_c 满足：其整体能量占比位于一个受上下界约束的低方差区域内。

（3）k 的作用

参数 k 的作用不是绝对硬性的“必须选满 k 个方向”，而是在不突破上界约束的前提下，尽量保证所选方向数不少于 k。若当前上界预算不足以继续加入更多方向，则即使尚未达到 k 个方向，也会提前停止。

因此，实际选出的方向数由数据分布与参数共同决定，并不固定等于 k。

（4）特殊情况

若某一类别特征值总和非正，或经过上述规则后没有任何方向被选中，则该类别的困难方向集合视为空集，此时 DDS 原始分数记为 0。

4.3.5 样本原始分数定义

设类别 c 最终选出的困难方向集合为
U_c = {u_j}_{j∈I_c}。

对于类别 c 中任一样本 x_i，其原始 DDS 分数定义为该样本在所有困难方向上的投影绝对值之和：
DDS_raw(i) = ∑_{j∈I_c} |<z_i - μ_c, u_j>|。

该定义反映了样本沿类内低方差困难方向的总体偏移幅度。若一个样本在这些方向上的偏移较大，则说明它偏离了类别内部最常见的主流变化模式，更可能对应少见结构、局部异常模式或与决策边界相关的变化，因此其 DDS 原始分数更高。

若某类别没有可用的困难方向，则该类别所有样本的
DDS_raw(i) = 0。

4.3.6 类内分位点归一化

由于不同类别在特征尺度、类别内部紧致程度以及有效困难方向数量方面可能存在差异，因此原始分数
DDS_raw(i)
不能直接跨类别比较。为此，本研究对每个类别分别进行分位点归一化。

对于类别 c，收集该类所有样本的原始分数集合
{DDS_raw(i) | y_i = c}。

记该集合的低分位点与高分位点分别为
q_c^{low} = Quantile({DDS_raw(i)}, 0.002),
q_c^{high} = Quantile({DDS_raw(i)}, 0.998)。

则样本 x_i 的最终 DDS 分数定义为
DDS(i) = clip((DDS_raw(i) - q_c^{low}) / (q_c^{high} - q_c^{low}), 0, 1), 当 y_i = c。

其中 clip(·, 0, 1) 表示将结果截断到 [0, 1] 区间内。

若某一类别满足
q_c^{low} ≈ q_c^{high}，
说明该类别样本在困难方向上的区分度很弱，或者所有样本的原始 DDS 值几乎一致。此时，为避免数值不稳定，统一将该类别样本的最终 DDS 分数置为 0.5，表示其中性状态。

4.3.7 指标含义与作用

按照上述定义，DDS(i) 越大，表示样本在所属类别的低方差困难方向上偏移越明显，越可能包含少见但有判别意义的局部结构变化；DDS(i) 越小，则表示样本主要位于类别内部更常见、更主流的变化模式附近。

因此，DDS 所强调的不是样本在类别中的“代表性中心位置”，而是样本是否承载了类内非主流但可能重要的困难信息。与 SA 关注语义对齐、Div 关注类内覆盖范围不同，DDS 更关注类内分布中那些不占主流但可能影响分类性能的细粒度变化方向，从而为静态综合评分补充另一类互补信息。

4.3.8 实现说明

特征空间：
DDS 使用 CLIP 图像编码器提取特征，并通过训练后的图像侧 adapter 进行映射后，再在该特征空间中进行类内 PCA 分析。这样可以在保留预训练语义结构的同时，使类内统计结构更贴近当前数据集。
方向排序方式：
特征值采用从小到大的顺序进行处理，因此被优先考虑的是类内低方差方向，而不是主方差方向。
参数含义：
k 控制希望保留的最少方向数；
eigval_lower_bound 用于避免把累计能量过低、过于极端的最小特征值方向直接纳入困难方向集合；
eigval_upper_bound 用于限制困难方向集合的总能量占比，使其始终处于低方差区域内；
pca_cov_reg 用于增强协方差矩阵特征分解的数值稳定性。
默认参数：
当前默认设置为
k = 5，
eigval_lower_bound = 0.02，
eigval_upper_bound = 0.2，
pca_cov_reg = 1e-6。
样本数过少时的处理：
若某类别样本数不超过 1，则无法构造有效协方差矩阵，此时该类别样本的 DDS 原始分数直接记为 0。

4.3.9 小结

DDS 指标从“类内低方差困难方向”的角度刻画样本质量，其基本思想是：类别内部最常见的变化模式往往由主方差方向描述，而那些方差较小但仍保有一定结构能量的方向，更可能对应少见、细粒度、潜在困难的变化因素。通过在这些方向上衡量样本偏移幅度，DDS 能够识别出那些虽然不典型、却可能对分类决策与泛化能力具有重要作用的样本。该指标与 SA、Div 在关注重点上互补，共同构成静态样本评估框架中的三个基础分量。

4.4 整体静态特征向量

f_i = [ SA_i, Div_i, DDS_i ]^T

将作为后续动态权重学习的输入。

5 基于交叉验证动态信息的静态评分权重学习

5.1 问题动机

前一章给出了 SA、Div、DDS 三个静态质量分量。它们分别从语义对齐、类内覆盖与类内低方差关键变化方向三个角度描述样本质量，但“这三个分量在当前任务中应当如何组合”并不是一个能够凭经验固定回答的问题。不同数据集、不同类别组织方式以及不同语义表征空间下，三个静态分量对数据选择的贡献强弱并不相同。若直接采用平均加权或人工指定权重，虽然可以得到一个可运行的评分函数，但这种做法缺少来自任务本身的依据；若依赖大量超参数搜索，则代价较高，也不利于方法的可解释性与复用性。

本项目采用的基本思路是：不直接用训练动态作为部署阶段的评分来源，而是把训练动态当作一次离线的监督信号，用它来指导静态分量之间的组合关系学习。也就是说，训练动态承担“教师”的角色，静态指标承担“学生”的角色。先通过代理模型的交叉验证训练日志，为每个样本构造一个连续的动态效用标签 u_i；再用静态特征向量
f_i = [SA_i, Div_i, DDS_i]^T
去回归该动态效用标签；最后得到一个只依赖静态特征、可直接作用于未见有标签样本的评分函数。

从方法目标看，这里的动态监督并不是为了直接判断“某个样本在当前代理模型下是否好学”，而是为了刻画“什么样的样本更可能在数据选择中发挥正面作用”。因此，动态分量本身应当满足三个要求。第一，单个分量内部语义应尽量纯粹，一个分量只表达一种偏好，而不把吸收性、边界性、互补性、覆盖性和噪声抑制等多种偏好混在一起。第二，不同分量之间应当真正互补，而不是形式不同但最后都在奖励同一类样本。第三，训练视角与验证视角的身份应严格区分：训练视角的分量主要描述“样本如何参与训练过程”，验证视角的分量主要描述“训练中形成的作用是否在 OOF 预测语境下仍能被观察到”。

基于上述考虑，当前版本不再沿用早期方案中较多、较杂的动态分量设计，而是统一为四个分量：吸收增益分量 A（Absorption Gain）、混淆互补分量 C（Confusion Complementarity）、迁移对齐分量 T（Transferability Alignment）与持久困难分量 P（Persistent Difficulty）。其中 A 与 C 来自训练视角，T 与 P 来自验证视角。A 强调样本在训练前中期是否被有效吸收，C 强调样本在类内是否对应较少见且互补的混淆模式，T 强调训练中样本推动的边界改进是否与同类验证集整体改善同步，P 强调样本在 OOF 验证语境下中后期是否仍持续表现出相对更高的困难性。最终，这四个分量共同构成动态监督标签的来源，再回归到静态特征 [SA, Div, DDS] 上，学习任务相关但仍保持静态可用的评分权重。

5.2 代理模型、日志记录与样本身份

5.2.1 代理模型与交叉验证

动态监督来自代理模型的 K 折交叉验证训练。设原始训练集为
D = {(x_i, y_i)}_{i=1}^N，
第 f 折包含训练子集 I_train^f 与验证子集 I_val^f，满足
I_train^f ∪ I_val^f = {1, …, N},    I_train^f ∩ I_val^f = ∅。
代理模型本身并不是后续部署时真正使用的评分模型，它的作用是产生具有训练语境与验证语境的轨迹信息。当前实现中，每一折都会保存训练子集与验证子集在各个 epoch 上的 logits 轨迹，从而为后续的四个动态分量提供统一输入。

交叉验证的关键价值不在于提高代理模型本身的精度，而在于引入 OOF（out-of-fold）预测轨迹。训练视角分量依赖样本作为训练样本时的学习过程；验证视角分量依赖样本作为验证样本时的预测过程。若没有 K 折交叉验证，验证视角分量便无法在不使用测试集的前提下获得真正的“未参与该折训练”的轨迹。

5.2.2 按折保存的最小必要日志

对每一折 f，至少保存以下内容：

(1) train_indices：该折训练子集的全局索引；
(2) val_indices：该折验证子集的全局索引；
(3) train_logits：形状为 [E, |I_train^f|, C] 的训练 logits 轨迹；
(4) val_logits：形状为 [E, |I_val^f|, C] 的验证 logits 轨迹。

其中 E 为训练轮数，C 为类别数。标签 y_i 不必显式重复存储，可以通过全局索引从原始数据集中恢复。loss、correct、margin 等量也不必单独写入日志，因为它们都可以由 logits 与标签在后处理中计算得到。这样做可以显著减小日志体积，并使所有动态分量共享同一份基础记录。

5.2.3 训练视角与验证视角的样本身份

四个动态分量所面对的“样本身份”并不相同，这一点必须在设计上明确区分。

（1）训练视角分量 A、C  
样本 i 在 K−1 个折中会作为训练样本出现，因此它拥有 K−1 条训练轨迹。对每个训练视角分量，先在每一折内计算该折给出的原始值，再在折间对这些值进行聚合，从而得到全局的 A(i) 与 C(i)。

（2）迁移对齐分量 D  
D 的目标样本同样是“作为训练样本出现的样本”。不过，它的定义并不只依赖该样本自身的训练轨迹，还依赖同类验证集整体改善曲线。因此，样本 i 也会在 K−1 个折中得到 {D_f(i)}，随后同样做折间聚合。

（3）验证视角分量 E  
样本 i 只会在一个折中作为验证样本出现，因此它只有唯一一条 OOF 验证轨迹。E 直接在这条轨迹上计算，并写回样本的全局索引位置，不需要做折间聚合。

5.2.4 时间窗口划分

当前代码实现统一采用三段式时间窗口划分。设总 epoch 数为 E_total，则按固定比例切分为：

T_early：前 30% 的 epoch；
T_mid：中间 40% 的 epoch；
T_late：后 30% 的 epoch。

当总轮数较少时，采用安全截断，保证三段窗口都非空。该划分方式的考虑如下：

（1）训练早期更适合刻画“吸收”而不是“困难”；  
（2）中期开始，模型已经脱离纯随机状态，可以观察更稳定的混淆模式；  
（3）后期更适合刻画迁移性与持久困难，但若只看最后少量 epoch，分量容易失活，因此对某些验证分量采用 mid+late 的更宽窗口。

当前版本中，A 主要使用 early 与 mid，C 使用 early 与 mid 的混淆变化，D 使用 mid+late 上的时序对齐，E 使用 mid+late 上逐时刻标准化后的困难统计。

5.3 动态分量设计的总体原则

在进入具体分量定义之前，先明确当前方案的设计原则。

第一，单分量语义纯粹。A 只刻画“可吸收且后期不过度波动”，不负责边界推进、互补性或验证困难；C 只刻画“类内混淆模式的互补性”，不负责样本是否容易学；D 只刻画“训练推动是否与验证改善同步”；E 只刻画“验证语境下的持续困难性”。这样设计的目的是减少分量间隐式重复与相互抵消。

第二，训练视角与验证视角的身份严格区分。A、C 只利用样本作为训练样本时的轨迹；D 虽然目标样本仍是训练样本，但其监督对象来自同类验证集整体改善曲线；E 则完全使用样本自身的 OOF 验证轨迹。当前版本明确不允许把验证分量简单写成“训练分量换个窗口或换个 logits 变换”。

第三，分量需要服务于数据选择，而不是只服务于“语义清晰”或“容易学习”。例如，A 用于避免把明显不可吸收的异常样本抬高，但它不应独占对“好样本”的解释权；C 与 D 的存在正是为了引入类内互补性与迁移性；E 的存在则是为了保留那些虽然在验证中仍持续困难、却可能对决策边界和泛化能力有正面意义的样本。

第四，分量定义尽量简单、可解释、少调参。当前版本固定采用 30%/40%/30% 的窗口划分，验证分量在综合效用标签中采用 0.5 的较小权重，原因在于每个样本只拥有一条验证视角轨迹，而训练视角分量可以在 K−1 个折上反复观测。

5.4 吸收增益分量：Absorption Gain（A）

5.4.1 定位与目标

A 用于刻画样本在训练视角下是否能够在前中期被模型有效吸收，并在后期不出现过大的概率波动。它强调的是“吸收增益”，而不是“样本先天就容易”。换句话说，A 并不直接奖励 early 时刻真实类概率本来就高的样本，而是奖励那些在 early 到 mid 阶段表现出明显吸收进展、同时在 late 阶段不过度抖动的样本。

这一分量的意义在于：训练过程中真正有用的样本，不应只是“从一开始就非常简单”，更应表现为模型在训练推进中能够逐步把它学进去。若样本在前中期没有明显吸收，或者在后期仍表现出高度不稳定，那么它对数据选择的正面意义通常更值得怀疑。

5.4.2 输入与统计量

对第 f 折中作为训练样本出现的样本 i，记其真实类概率为
p_{i,y,t}^{train} = softmax(z_{train}^f[t, i])_{y_i}。

分别定义三段窗口上的统计量：
μ_i^{early,f} = mean_{t∈T_early} p_{i,y,t}^{train},
μ_i^{mid,f}   = mean_{t∈T_mid}   p_{i,y,t}^{train},
v_i^{late,f}  = Var_{t∈T_late}   p_{i,y,t}^{train}。

其中 μ_i^{mid,f} − μ_i^{early,f} 表示该样本从 early 到 mid 的真实类概率吸收增益，v_i^{late,f} 表示 late 阶段是否仍有较大抖动。

5.4.3 折内定义

对同一折内所有训练样本，分别对增益项与方差项做折内标准化：
z_gain^f(i) = z( μ_i^{mid,f} − μ_i^{early,f} ),
z_var^f(i)  = z( v_i^{late,f} ).

然后定义该折内的吸收增益原始值为
A_raw^f(i) = z_gain^f(i) − 0.5 · z_var^f(i).

这里将 late 方差项的系数固定为 0.5，而不是 1。这样做的考虑是：A 的主体应当强调前中期吸收增益，后期波动只作为辅助约束，而不应过度压制那些在前中期确有明显吸收、但后期仍存在一定正常波动的样本。

5.4.4 折间聚合与归一化

样本 i 会在 K−1 个折中作为训练样本出现，因此得到 {A_raw^f(i)}。记折间聚合操作为 Agg(·)，当前默认取中位数，则
A_raw(i) = Agg_f A_raw^f(i).

再对全局 A_raw(i) 做鲁棒分位点 min-max 归一化，映射到 [0,1]，得到最终 A(i)。

5.4.5 直觉解释

A 高的样本通常满足：前中期真实类概率上升明显，说明样本并非只是“容易”，而是能够被训练有效吸收；同时 late 阶段概率抖动不至于过大，说明该样本的学习结果更稳定。反之，若样本前中期没有明显吸收，或 late 阶段仍高度波动，则 A 会下降。

5.5 混淆互补分量：Confusion Complementarity（C）

5.5.1 定位与目标

C 用于刻画样本在训练视角下是否对应一种类内较少见、具有互补性的 rival confusion 变化模式。与 A 不同，C 不再关心样本是否容易被吸收，而是关心该样本在训练推进中“主要压制了哪一类错误竞争关系”。若这种竞争关系在同类样本中较为少见，则样本可能为数据子集提供了额外的互补信息。

该分量之所以重要，是因为数据选择不仅要考虑单样本自身是否好学，也要考虑样本在集合层面上是否补足了类内分布中较少出现的变化模式。C 正是为此而设计。

5.5.2 混淆分布与变化向量

对训练样本 i，在 epoch t 的 softmax 概率向量记为 p_{i,t}^{train}。去掉真实类分量，并对其余类别重新归一化，得到混淆分布 q_{i,t}^{train}。若真实类标签为 y_i，则
q_{i,t}^{train}(k) =
p_{i,t}^{train}(k) / Σ_{j≠y_i} p_{i,t}^{train}(j),    k ≠ y_i，
并令 q_{i,t}^{train}(y_i)=0。

接着在 early 与 mid 两段上分别取均值：
\bar q_i^{early,f} = mean_{t∈T_early} q_{i,t}^{train},
\bar q_i^{mid,f}   = mean_{t∈T_mid}   q_{i,t}^{train}。

定义混淆变化向量：
Δq_i^f = \bar q_i^{early,f} − \bar q_i^{mid,f}。

它表示训练推进过程中，该样本主要压低了哪些 rival class 的混淆概率。

5.5.3 类内 kNN 互补性

固定类别 c，在同一折训练子集内收集该类所有样本的 {Δq_i^f}。对其中样本 i，在同类样本内部计算 k 近邻平均距离：
d_i^f = mean_{j∈kNN_c(i)} ||Δq_i^f − Δq_j^f||_2。

定义该折内的混淆互补分量原始值为
C_raw^f(i) = d_i^f。

若类内样本数不足，则自动退化为使用可用邻居数，并在极端情况下置为 0。当前设计不再把 gap、loss、entropy 等其他量混入 C 中，从而保持其语义纯粹：C 只刻画“类内混淆变化模式是否互补”。

5.5.4 折间聚合与归一化

与 A 相同，样本 i 会在 K−1 个折中作为训练样本出现，因此得到 {C_raw^f(i)}。折间聚合得到
C_raw(i) = Agg_f C_raw^f(i)，
随后做鲁棒分位点归一化，得到最终 C(i)。

5.5.5 直觉解释

若某个样本在训练推进中压制的是类内较少见的一类竞争关系，则其 Δq 模式会与同类大多数样本不同，因此类内 kNN 距离较大，C 较高；若样本所对应的混淆变化只是同类中常见模式的重复，则 C 较低。该分量因此更偏向“集合互补性”而非“单样本可学习性”。

5.6 迁移对齐分量：Transferability Alignment（T）

5.6.1 定位与目标

T 用于刻画：样本在训练视角下推动的边界改进，是否与同类验证集整体改善曲线同步。它并不直接看样本自身在验证轨迹上是否表现得更好，而是看“该样本推动训练的方式”能否与“同类验证集真正受益的时间位置”对齐。

从数据选择角度看，这个分量的作用是把“训练中似乎有用”与“验证中同类真的同步受益”联系起来。若一个训练样本的推进主要发生在那些验证端同类也同步改善的 epoch，则说明它更可能对应真正可迁移的训练贡献，而不是局限在该样本自身的局部现象。

5.6.2 验证侧类均值改善曲线

在第 f 折的验证子集上，先由 logits 得到真实类概率：
p_{x,t}^{val} = softmax(z_{val}^f[t, x])_{y_x}。

定义验证端平滑损失：
ℓ_{val}^f(t, x) = log( 1 − log p_{x,t}^{val} ) 这一写法在实现中等价于对负对数似然作对数压缩，
更直接的连续写法为
ℓ_{val}^f(t, x) = log(1 + (−log p_{x,t}^{val}))。

然后对每个类别 c 构造验证端类均值曲线：
V^f(t,c) = mean_{x∈I_val^f, y_x=c} ℓ_{val}^f(t,x)。

再定义正向改善量：
ΔV^f(t,c) = max(0, V^f(t−1,c) − V^f(t,c)),    t ≥ 2。

ΔV^f(t,c) 越大，表示验证集中类别 c 在 epoch t 附近的整体表现有更明显的改善。

5.6.3 训练侧样本推进曲线

对第 f 折中的训练样本 i，定义训练端 margin：
m_{i,t}^{train} = z_{train}^f[t,i,y_i] − max_{k≠y_i} z_{train}^f[t,i,k]。

再定义一阶差分：
Δm_{i,t}^{train} = m_{i,t}^{train} − m_{i,t−1}^{train}。

为避免离散抖动和极端负值的影响，只保留正向推进并作 softplus 平滑：
g_{i,t}^{train} = τ_D · softplus( Δm_{i,t}^{train} / τ_D ).

其中 τ_D 为平滑温度，决定“推进速度”被如何平滑地映射为非负值。

5.6.4 mid+late 上的同步性定义

当前版本不再在全时段上做相关，而是只在
T_align = T_mid ∪ T_late
上计算训练侧推进与验证侧改善的同步性。对样本 i，令 c = y_i，记
a_i^f = {g_{i,t}^{train}}_{t∈T_align},
b_c^f = {ΔV^f(t,c)}_{t∈T_align}。

定义该折内的迁移对齐原始值：
T_raw^f(i) = corr(a_i^f, b_c^f)。

实现上可用 Pearson 相关，也可以在必要时退化为余弦相似度。若相关值定义在 [−1,1]，则再映射到 [0,1]：
T_map^f(i) = (T_raw^f(i) + 1) / 2。

5.6.5 折间聚合与归一化

样本 i 会在 K−1 个折中作为训练样本出现，因此得到 {T_map^f(i)}。折间聚合得到
T_raw(i) = Agg_f T_map^f(i)，
随后再做鲁棒归一化，得到最终 T(i)。

5.6.6 直觉解释

若样本 i 在训练中产生明显正向 margin 推进，而这些推进主要出现在“同类验证集也同步改善”的时期，则 T 较高；若样本 i 的推进与验证端改善节奏几乎无关，或者训练推进发生在验证端并无对应受益的时段，则 T 较低。该分量因此强调“训练贡献的可迁移性与时序对齐”。

5.7 持久困难分量：Persistent Difficulty（P）

5.7.1 定位与目标

P 用于刻画样本在 OOF 验证语境下，中后期是否仍持续表现出相对更高的困难性。这里的“困难”并不是指简单地“没学会”，也不是通过早期真实类概率再去扣减一个基线；否则会与 A 产生信息重叠，并在聚合时互相抵消。当前版本重新回到“margin 困难 + 不确定性困难”这两个通道的思想，但不再直接把原始难度跨 epoch 平均，而是采用“逐时刻标准化，再跨时刻累加”的统计方式，以修复尺度失衡导致的大量样本失活问题。

这一分量的直觉是：如果在某一时刻的验证样本群体中，一个样本的 margin 困难和不确定性都显著高于同一时刻的其他验证样本，那么它在该时刻更接近“相对持续困难”的状态。若这种相对困难性在 mid+late 上不断出现，则 E 会升高。

5.7.2 输入与时间窗口

对样本 i，只使用其所在折的 OOF 验证 logits 轨迹 z_{val}[t,i,:] 与标签 y_i。E 使用的时间窗口为
T_hard = T_mid ∪ T_late，
而不是只用 late。这样做的原因是：前期的不稳定性已由 A 处理，中后期既包含比 late 更丰富的信息，又仍处在“困难性”可以被合理解释的阶段。

5.7.3 逐时刻 margin 困难与熵困难

对 epoch t，定义验证 margin：
m_{i,t}^{val} = z_{val}[t,i,y_i] − max_{k≠y_i} z_{val}[t,i,k]。

构造该时刻的原始 margin 困难量：
r_{i,t}^{margin} = − m_{i,t}^{val}。

margin 越小，乃至为负，表示越接近边界甚至被竞争类压制，因此 r_{i,t}^{margin} 越大，困难性越强。

再由 logits 计算 softmax 概率 p_{i,t}^{val}(·)，定义验证熵：
H_{i,t}^{val} = − Σ_c p_{i,t}^{val}(c) log( p_{i,t}^{val}(c) + ε )。

构造归一化的原始熵困难量：
r_{i,t}^{entropy} = H_{i,t}^{val} / log C。

该量越大，表示该样本在该时刻越不确定。

5.7.4 逐时刻 fold 内标准化

E 的关键修改不在于换掉上述两个困难来源，而在于改变统计尺度。对于固定的 epoch t，在同一折的全部验证样本上，分别对两个原始困难量做标准化：

z_{i,t}^{margin} =
( r_{i,t}^{margin} − μ_t^{margin} ) / ( σ_t^{margin} + ε ),

z_{i,t}^{entropy} =
( r_{i,t}^{entropy} − μ_t^{entropy} ) / ( σ_t^{entropy} + ε ).

其中 μ_t 与 σ_t 都是在“该折、该 epoch、全部验证样本”上的统计量。这样处理后，P 不再依赖某个 epoch 上原始 margin 或熵的绝对尺度，而是关注“该样本相对于同一时刻其他验证样本是否更难”。这一步正是为了解决旧版 E 在 CIFAR-10、CIFAR-100 上大量样本堆积到 0 附近的问题。

5.7.5 中后期聚合

在 T_hard = T_mid ∪ T_late 上，分别对逐时刻标准化后的结果做平均：
P_i^{margin}  = mean_{t∈T_hard} z_{i,t}^{margin},
P_i^{entropy} = mean_{t∈T_hard} z_{i,t}^{entropy}。

最终定义原始持久困难分量为
E_raw(i) = P_i^{margin} + P_i^{entropy}。

这里采用求和而不是再引入额外门控或早期基线扣减，原因在于：P 的任务仅是刻画“验证视角的相对持续困难性”，不应显式兼顾吸收性、噪声抑制等其他语义。

5.7.6 归一化与直觉解释

P_raw(i) 计算完成后，再沿用统一的鲁棒分位点归一化，得到最终 P(i)。

P 高的样本通常满足：在 mid+late 的多个时刻里，无论从 margin 难度还是不确定性难度看，它都持续高于同一时刻的多数验证样本。P 因此并不是在鼓励“完全学不会”的样本，而是在保留那些在验证语境下相对更难、但这种困难性确实持续存在的样本。

5.8 动态效用标签的构造

5.8.1 四个分量的组合

得到 A、C、T、P 四个分量后，需要进一步构造统一的动态效用标签。当前版本采用如下加权和：

u_raw(i) = A(i) + C(i) + 0.5 · T(i) + 0.5 · P(i)。

之所以给验证视角分量 T、P 较小的系数 0.5，而训练视角分量 A、C 保持系数 1，主要有两个原因。

第一，训练视角分量可以在 K−1 个折中多次观察同一样本，统计更充分；验证视角中，每个样本只有一条 OOF 轨迹，因此分量的观测次数与稳定性相对更弱。第二，A 与 C 分别从吸收性和类内互补性两个方面提供了训练语境中的主监督，而 T 与 P 更像对迁移性和持续困难性的补充监督。采用 1 : 1 : 0.5 : 0.5 的配比，有助于让验证视角分量发挥修正作用，而不至于主导整个动态效用标签。

5.8.2 最终归一化

设总权重为
w_sum = 1 + 1 + 0.5 + 0.5 = 3。

则动态效用标签定义为
u(i) = u_raw(i) / w_sum。

当前实现中，u 已经是基于 [0,1] 范围分量组合得到的连续值，可直接作为后续静态权重学习的监督标签。为了与已有读取流程兼容，也会同时缓存各个分量的 raw / fold-normalized / aggregated / final-normalized 结果以及最终的 u_raw、u_scores、u_norm。

5.9 基于静态特征的权重学习

5.9.1 静态特征向量

对样本 i，静态特征向量定义为
f_i = [SA_i, Div_i, DDS_i]^T。

它们来自前一章定义的统一语义表征空间与对应静态指标计算过程。需要强调的是，动态监督标签 u 与静态特征 f_i 的角色不同：u 由代理模型交叉验证动态信息给出，f_i 由 CLIP + Adapter 构造的静态表征空间给出。最终部署时，真正保留下来的是静态评分函数，而不是动态轨迹本身。

5.9.2 约束岭回归

当前版本采用带非负约束的岭回归学习静态分量权重。设训练样本矩阵为
F = [f_1^T; ...; f_N^T] ∈ R^{N×3}，
监督标签为
u = [u(1), ..., u(N)]^T ∈ R^N。

目标是求解
min_{w,b} ||Fw + b·1 − u||_2^2 + λ ||w||_2^2
s.t.  w_j ≥ 0,  j = 1,2,3。

其中
w = [w_SA, w_Div, w_DDS]^T
为静态分量权重，b 为偏置项，λ 为岭正则系数。非负约束的考虑在于：三个静态分量都被视为“正面质量分量”，因此在当前框架中不允许学出符号相反的解释。

5.9.3 按 seed 分别学习权重

虽然动态监督标签 u 来自同一份 proxy_training_seed 下的交叉验证动态日志，但静态特征 SA、Div、DDS 并不是在所有实验 seed 下完全相同的。原因在于：图像端与文本端 Adapter 的训练会受到随机种子影响，从而进一步影响 SA、Div、DDS 的数值分布。因此，权重学习过程不能简单地“动态标签算一份、静态特征也算一份、然后把同一份权重复制给所有 seed”。

当前实现采用如下处理方式：

（1）动态分量 A、C、T、P 及其最终监督标签 u 只计算一份，共享同一个 proxy_training_seed；  
（2）对每个输出 seed，分别加载该 seed 训练得到的 image adapter 与 text adapter；  
（3）分别计算该 seed 下的 SA / Div / DDS；  
（4）用同一份动态标签 u，分别回归出每个 seed 专属的静态权重
w^{(seed)} = [w_SA^{(seed)}, w_Div^{(seed)}, w_DDS^{(seed)}]^T；
（5）将每个 seed 的结果分别写入 scoring_weights.json。

因此，当前框架中真正共享的是“动态监督标签”，而不是“学习出的静态权重”。

5.9.4 诊断信息

为了检查动态监督与静态分量之间的关系，当前实现还会按 seed 输出四个动态分量与三个静态指标之间的 Pearson 相关系数矩阵，即：
corr(A, SA), corr(A, Div), corr(A, DDS),
corr(C, SA), corr(C, Div), corr(C, DDS),
corr(T, SA), corr(T, Div), corr(T, DDS),
corr(P, SA), corr(P, Div), corr(P, DDS)。

这些诊断量本身并不参与权重优化，但它们有助于分析某一动态分量是否在统计上过于贴近某一个静态指标，或者某个分量是否已经出现明显失活。类似地，学习出的权重若长期表现为 SA 显著高于 1/3、或 Div 长期垫底，也可以反过来帮助判断动态分量设计是否遗漏了某种对数据选择有价值的样本属性。

5.10 最终静态评分函数

在得到 seed 专属的静态权重 w 与偏置 b 之后，最终静态评分函数可写为
Score(i) = w_SA · SA_i + w_Div · Div_i + w_DDS · DDS_i + b。

该评分函数完全不再依赖训练动态；训练动态只在离线阶段承担监督与校准作用。由此，整个方法实现了从“训练过程中的样本价值判断”到“可直接作用于未见有标签样本的静态评分”的转换。对已有训练集中的样本，可据此直接排序选择；对新到达的有标签样本，也可在不重新运行代理模型的情况下快速给出任务相关的质量评分。

5.11 本章小结

本章给出了当前版本基于交叉验证动态信息学习静态评分权重的完整设计。其核心不在于构造一个越来越复杂的动态打分器，而在于：先把训练动态拆解为吸收性、类内互补性、迁移对齐性和持久困难性四个语义尽量纯粹的分量，再把这些分量组合为统一的动态效用标签，最后回归到 SA、Div、DDS 三个静态指标上，得到一个可复用的静态评分函数。

从结构上看，A 与 C 对应训练视角，T 与 P 对应验证视角；A 强调吸收，C 强调互补，T 强调迁移同步，P 强调相对持续困难；四者并不追求从不同形式上重复表达同一类样本，而是共同服务于“什么样的样本更可能对数据选择有正面作用”这一问题。通过这一设计，训练动态被保留为一次性监督信号，而最终对外提供的是任务相关但仍保持静态可用性的样本质量评分。

6 最终评分函数与未见样本评估

6.1 最终评分定义

对任意样本 x（训练内或训练外），最终样本质量评分定义为：

Score(x) = ẇ_1 · SA(x) + ẇ_2 · Div(x) + ẇ_3 · DDS(x)

其中：

SA(x)：用 CLIP + Adapter 后特征 g(x) 与类别文本 t_y 构造的语义对齐度（实际使用时按前文定义的 margin 形式计算）；

Div(x)：在 g(x) 所在类别的特征空间中，用 kNN 距离 rank 定义的相对稀疏度；

DDS(x)：在该类别基于 g(·) 做 PCA 得到的低方差方向上的投影幅度；

ẇ：前一阶段从训练动态学得的固定权重。

6.2 未见样本评分

对未见样本 x_new：

6.2.1 用 CLIP 图像编码器和图像端 Adapter 得到 g_new
6.2.2 用英文类别 prompt 经文本编码器与文本端 Adapter 得到 t̃_{y_new}，并计算 SA_new
6.2.3 在所需位置对 g_new 与相关类别特征做显式 L2 归一化，再将 g_new 插入对应类别的 G_c，计算 d_new → rank_new → Div_new
6.2.4 用同类别的 μ_c 和 PCA 方向 u_k，计算 DDS_new
6.2.5 用固定 ẇ 计算

Score(x_new) = ẇ_1 SA_new + ẇ_2 Div_new + ẇ_3 DDS_new

整个过程不依赖训练动态，也不需重新训练模型。

7 基于 Score 的数据选择与 Selection Optimization

7.1 直接基于 Score 的排序选择

最直接使用方式：

对训练集中每个样本计算 Score(i)；

按 Score 排序，选取 top-k% 子集参与训练。

这实现了一个简单、直接的“样本质量驱动的数据裁剪”：高分样本保留、低分样本舍弃。

7.2 group 模式——按类别逐轮贪心添加的子集构造方法

为使数据选择过程与前文“静态指标评分 + 动态监督学习权重 + 下游应用”的主线保持一致，本文不再将 group 模式设计为一个带全局交换、后验修正项和额外筛选规则的启发式搜索器，而是将其重构为一种按类别分解、逐轮增量添加样本的子集构造方法。该方法的核心思想是：在类别层面显式控制预算分配，从构造机制上消除类别数量失衡问题；在样本层面则统一依据“静态质量 + 当前子集上的动态多样性 + 分布保真约束”进行评分，从而使 group 模式真正成为评分框架在数据选择场景下的组合式扩展，而不是脱离评分主线的独立算法。

7.2.1 设计思想

topk 模式依据全集上预先计算好的单样本综合分数直接做类内截断，优点是简单、稳定，但缺点也很明确：它默认每个样本的价值是相互独立的，无法刻画“一个样本在当前已选子集背景下是否仍有补充价值”。特别是在低 keep ratio 场景下，topk 容易因为连续选择一批高静态分样本而使类别中心发生显著偏移，进而导致子集在保持代表性方面出现问题。

基于这一观察，group 模式不再试图通过全局交换搜索一个难以解释的“最优子集”，而是从更贴合本项目设计理念的角度出发，直接构造一个子集：

（1）样本自身的基础质量仍由静态指标决定；
（2）样本是否值得在当前时刻加入，还要看它相对于当前类别子集的动态多样性价值；
（3）样本加入后不应使当前类别子集的分布中心偏离该类别原始分布中心太多。

因此，group 模式本质上是在每个类别内部不断回答这样一个问题：在当前已经选出的该类样本基础上，下一步应该再补入哪一个样本，才能兼顾静态质量、当前覆盖增益以及分布中心保真性。

7.2.2 类别预算与构造原则

设原始训练集为
D = igcup_{c=1}^{C} D_c,
其中 D_c 表示类别 c 的样本集合，|D_c| 表示该类别样本数。给定保留比例 keep ratio，记
sr = keep\_ratio / 100.

目标总保留样本数为
N_{tar} = round(sr \cdot |D|).

为避免在优化目标中再额外引入类别平衡惩罚项，group 模式直接在构造阶段为每个类别分配预算。对类别 c，先计算实值预算：

b_c^{raw} = sr \cdot |D_c|.

然后取其整数下界：

b_c^{floor} = \lfloor b_c^{raw} 
floor.

若 \sum_c b_c^{floor} < N_{tar}，则将剩余预算按小数部分

r_c = b_c^{raw} - b_c^{floor}

从大到小依次分配，直到所有类别预算之和恰好等于 N_{tar}。于是得到每个类别的最终整数预算 b_c。该分配方式保证：

（1）各类别预算整体上遵循等比例缩放；
（2）所有类别预算之和严格等于目标保留样本数；
（3）不再需要在 group 模式中额外设计类别平衡罚项。

在后续构造过程中，只要某个类别已达到预算 b_c，就不再为其添加新样本；未达到预算的类别继续参与后续轮次。

7.2.3 初始化策略

由于构造初期当前类别子集为空，动态 Div 与分布约束的计算都不够稳定，因此 group 模式首先执行纯静态初始化。对每个类别 c，在该类别内部定义静态初始化分数：

Score_{init}(x_i) = w_{sa} \cdot SA_i + w_{dds} \cdot DDS_i.

其中 SA_i 和 DDS_i 分别是在全集上预先计算得到的静态语义对齐分数与难度方向分数，w_{sa} 与 w_{dds} 使用当前权重组中对应的权重。然后在类别 c 内选出静态初始化分数最高的前 3 个样本作为初始子集：

S_c^{(0)} = Top3\{ Score_{init}(x_i) \mid x_i \in D_c \}.

若某类别预算 b_c < 3，则仅初始化 min(3, b_c, |D_c|) 个样本。这样做的目的，是为后续的动态增量构造提供一个语义质量较高且难度信息较充分的起点，避免构造初期完全由分布约束或动态多样性项主导。

7.2.4 单步增量评分函数

设当前类别 c 的已选子集为 S_c，未选候选集合为

U_c = D_c \setminus S_c.

对任意候选样本 x \in U_c，若将其加入当前类别子集，则其当前轮次的综合加权分定义为：

Score_{add}(x \mid S_c)
= w_{sa} \cdot SA(x)
+ w_{dds} \cdot DDS(x)
+ w_{div} \cdot Div_{add}(x \mid S_c)
+ w_{dist}(kr) \cdot Dist_{add}(x \mid S_c).

其中：

（1）SA(x) 与 DDS(x) 为全集静态指标；
（2）Div_{add}(x \mid S_c) 为相对于当前类别子集 S_c 计算得到的动态多样性分数；
（3）Dist_{add}(x \mid S_c) 为候选样本加入后对子集分布中心保真性的改善分数；
（4）w_{sa}, w_{dds}, w_{div} 为当前权重组下的指标权重；
（5）w_{dist}(kr) 为分布约束项权重，它不参加静态权重学习，而是由 keep ratio 决定。

当前实现中，分布约束项权重采用随 keep ratio 线性衰减的形式：

w_{dist}(kr) = max(0, 1 - 0.01 \cdot kr).

因此：

kr = 20 时，w_{dist} = 0.8；
kr = 50 时，w_{dist} = 0.5；
kr = 90 时，w_{dist} = 0.1；
kr = 100 时，w_{dist} = 0。

这一设计反映了如下考虑：低 keep ratio 时子集规模很小，若缺乏分布约束，当前类别子集更容易因连续选入高静态分样本而偏离原始分布中心，因此需要更强的分布保真约束；随着 keep ratio 增大，子集本身已更接近原始数据分布，分布约束项应逐步减弱，以免 group 模式在高保留率下过度接近 herding。

7.2.5 动态多样性分数 Div_{add}

动态多样性分数用于刻画候选样本相对于当前类别子集的补充价值。与静态 Div 在全集内一次性计算不同，group 模式中的 Div 是在当前类别子集背景下动态重算的。

设当前类别 c 已选子集大小为 |S_c| = m_c。对每个候选样本 x \in U_c，使用当前类别子集 S_c 作为参考集合，在同一 CLIP 图像表征空间（经 image adapter 适配后）中计算其到参考集合的平均近邻距离。当前实现中，动态 Div 的近邻数不再直接沿用静态 Div 默认的比例参数，而是在每一步由外部显式设置为具体整数：

k_c^{dyn} = max\left(3, \left\lceil 0.05 \cdot m_c 
ight
ceil
ight).

即：动态 Div 的参考近邻数取当前类别已选样本数的 5%，向上取整，并强制至少为 3。当当前类别子集过小、可用参考样本数不足 k_c^{dyn} 时，则自动退化为使用当前全部已选样本作为参考集，平均时的分母也取当前实际参考样本数。这样既保持了 Div 指标“类内近邻平均距离”的原始定义，又使其在 group 构造的早期阶段仍然可以稳定计算。

对每个候选样本 x 的动态 Div 原始值记为 d_x^{dyn}。在当前轮该类别全部候选样本的动态 Div 原始值集合上，再做一次分位点 min-max 归一化：

Div_{add}(x \mid S_c)
= clip\left(
rac{d_x^{dyn} - q_{low}^{dyn}}{q_{high}^{dyn} - q_{low}^{dyn}},
0, 1

ight),

其中 q_{low}^{dyn} 与 q_{high}^{dyn} 分别取当前轮候选集原始值的 0.2% 与 99.8% 分位点。若两者几乎相同，则统一置为 0.5。这样处理后，Div_{add} 反映的是：在当前类别子集背景下，候选样本是否处于一个尚未充分覆盖、值得补入的位置。

7.2.6 分布约束分数 Dist_{add}

为防止 group 模式在低 keep ratio 下像 topk 一样连续选入一批高静态分样本，从而导致类别子集中心显著偏移，本文引入一个分布约束分量，用于衡量候选样本加入后对子集分布中心的改善程度。

设类别 c 在 CLIP 图像表征空间中的全集均值为

\mu_c^{full} = rac{1}{|D_c|} \sum_{x_i \in D_c} z_i,

其中 z_i 为样本在经 image adapter 适配后的图像特征。当前类别已选子集 S_c 的特征均值记为

\mu_c^{sub} = rac{1}{|S_c|} \sum_{x_i \in S_c} z_i.

对候选样本 x \in U_c，记加入后的新子集均值为

\mu_c^{new}(x) = rac{|S_c| \cdot \mu_c^{sub} + z_x}{|S_c| + 1}.

则该候选样本带来的分布中心改善量定义为：

\Delta_{dist}(x \mid S_c)
= \| \mu_c^{sub} - \mu_c^{full} \|_2
- \| \mu_c^{new}(x) - \mu_c^{full} \|_2.

若 \Delta_{dist} 越大，说明加入该样本后当前子集中心越接近原始类别中心，因此该样本在分布保真意义上的价值越高。对当前轮全部候选样本的 \Delta_{dist} 原始值同样做一次分位点 min-max 归一化，得到：

Dist_{add}(x \mid S_c)
= clip\left(
rac{\Delta_{dist}(x \mid S_c) - q_{low}^{dist}}{q_{high}^{dist} - q_{low}^{dist}},
0, 1

ight).

这样，Dist_{add} 就不再是一个后验整体惩罚项，而是成为每一步构造时直接参与候选评分的正向分量。它的作用不是替代动态 Div，而是在低 keep ratio 情况下抑制当前类别子集中心被过快拉偏。

7.2.7 权重使用方式

group 模式中，SA、DDS、Div 三项统一使用当前权重组对应的权重：

（1）在 naive 模式下，使用预设的均匀权重；
（2）在 learned 模式下，使用前文由训练动态监督学习得到的权重。

因此，group 模式下不同权重组会真正影响：

（1）初始化阶段的静态高分样本选择；
（2）后续每一步增量构造中的候选排序；
（3）最终得到的子集结构。

相比之下，分布约束项并不参与静态权重学习，而是由 keep ratio 决定其权重强度。这一处理方式使 group 模式能够在保持评分框架主线一致性的同时，针对低保留率下特有的分布偏移问题进行补偿。

7.2.8 候选池机制

在每一轮、每一个类别内部，对当前所有候选样本 x \in U_c 计算 Score_{add}(x \mid S_c)，再按得分从高到低排序。若只取排序第一的样本直接加入，则该过程是完全确定性的贪心添加。为了在保持高分导向的同时保留一定探索性，当前实现引入了一个很轻量的候选池机制。

设候选池大小为 p。则在当前类别中：

（1）先取综合得分最高的前 p 个候选样本构成候选池；
（2）若 p = 1，则直接选择该样本；
（3）若 p > 1，则从候选池中随机采样 1 个样本加入当前类别子集。

默认情况下 p = 1，因此当前 group 模式默认仍是一个严格贪心的逐类构造方法。候选池的作用主要是在需要时为实验提供一点受控随机性，以减轻早期单步决策错误对后续整个构造过程的连锁影响。

7.2.9 逐轮构造流程

综合上述设计，最新版 group 模式的整体构造流程如下。

（1）准备阶段
读取或计算全集上的 SA、DDS、静态 Div 分数，加载当前权重组的权重，提取所有样本在 CLIP + image adapter 特征空间中的图像特征，并预先计算各类别的全集特征均值。

（2）预算分配
根据 keep ratio 与各类别样本数，计算总保留样本数 N_{tar}，再按比例为每个类别分配整数预算 b_c。

（3）静态初始化
对每个类别 c，使用 Score_{init}(x)=w_{sa}SA(x)+w_{dds}DDS(x) 在该类中选出前 min(3,b_c,|D_c|) 个样本，构成初始子集 S_c^{(0)}。

（4）逐轮添加
对所有尚未达到预算的类别，按轮次依次执行以下操作：

  a. 计算当前类别的动态近邻数 k_c^{dyn}=max(3, ceil(0.05|S_c|))；

  b. 对该类每个未选候选样本计算动态 Div 原始值并归一化，得到 Div_{add}(x|S_c)；

  c. 对该类每个未选候选样本计算分布中心改善量并归一化，得到 Dist_{add}(x|S_c)；

  d. 按 Score_{add}(x|S_c) 对候选样本排序；

  e. 从前 p 个候选样本中选择 1 个加入当前类别子集。

若最后剩余预算不足以让所有未满类别都各再增加 1 个样本，则从未满预算的类别中随机选择若干个类别，各完成最后一次添加。

（5）终止与输出
当所有类别都达到预算后，合并所有类别子集得到最终 mask，并输出各类别选样数、当前构造过程中累计的贪心评分、最终子集在动态 Div 下的综合得分以及分布中心偏移程度等统计量。

7.2.10 方法特点与意义

总体而言，最新版 group 模式不再追求一个难以解释的“全局交换式最优子集”，而是将数据选择过程改写为一个与评分框架主线高度一致的、可逐步解释的构造过程。它具有以下特点：

第一，主线一致。SA 与 DDS 仍然承担样本静态质量评估的主干作用，动态 Div 仍沿用原框架中“类内覆盖度”的几何定义，只是将参考集合从全集切换为当前子集；分布约束项也建立在同一 CLIP 图像表征空间中，因此 group 模式内部各个分量的几何语义是一致的。

第二，类别控制内生化。通过预算分配与逐类添加，类别数量问题在构造阶段已被直接处理，不再需要在目标函数中额外加入类别平衡惩罚项，从而避免了旧版本中“评分主线”和“后验修正项”混杂的问题。

第三，低 keep ratio 更稳健。通过将分布约束项权重设置为 keep ratio 的函数，group 模式能够在低保留率下显式抑制子集中心漂移，在高保留率下又逐步减弱该约束，避免方法过度退化为 herding。

第四，可解释性更强。对于每一次样本加入，都可以明确解释为：该样本为什么在当前时刻值得加入，是因为它静态质量更高、当前覆盖增益更大、还是更有利于保持当前子集的分布中心。这种逐步可解释的构造方式更符合本项目“样本质量评估框架”的研究定位。

因此，最新版 group 模式的定位不是脱离评分框架另做一个强搜索算法，而是将“静态质量评估”推广为“带动态覆盖与分布保真约束的逐类增量子集构造方法”，是本项目从单样本评分走向集合式数据选择应用的自然延伸。
8 与直接使用训练动态评分的对比与方案意义

常见质疑：既然训练动态中已经有各种信息，为什么不直接把训练动态用作评分，而要多一层“静态指标 + 动态监督学习权重”的设计？

核心回答：

8.1 训练动态是“模型行为”，不是“样本固有属性”

换模型、换初始化、换优化器、换 batch 顺序，动态模式都会改变；

对未见样本没有训练轨迹，无法直接给分；

仅用训练动态无法定义一个可复用的“样本质量模型”。

8.2 静态指标是“样本固有属性”

基于 CLIP + Adapter 的语义对齐（SA）、类内相对稀疏度（Div）、类内难度方向（DDS）；

与训练过程无关，只与样本本身及其类别标签有关；

可以对任意新样本、任意新的训练过程复用。

8.3 我们的设计是：“训练动态做老师，静态评分函数做学生”

训练动态通过 u_i 告诉我们：在这个具体任务中，哪类样本更有价值；

静态指标通过回归学习 ẇ，从而捕捉到“哪些静态特征模式对应高价值样本”；

一旦学成，静态评分函数 Score(·) 就可以在没有训练动态的情况下独立使用。

8.4 Dataset Adapter 的引入进一步增强了这一框架

Adapter 先对 CLIP 特征进行数据集适配，使 SA / Div / DDS 的统计结构更加合理；

Adapter 训练一次后固定，不与动态权重学习形成循环依赖；

对未见样本仍然可以直接前向 through adapter，然后用统一的 Score(·) 打分。

8.5 应用场景广泛而不是“很小的角落”

自动构建和维护训练数据仓库：新增样本先打分，再决定是否加入训练集；

增强数据质量控制：对自动数据增强产生的样本、弱标或噪声数据进行质量筛选；

长期在线学习：在不断到来的数据流中有一个稳定的、与模型结构解耦的样本质量评价函数。

本项目交付的是一个“可解释、可泛化、可复用的样本质量评分函数”，而不仅仅是一套对某次训练日志的分析工具。

9 实现与实验规划简述

9.1 数据集

CIFAR-10 / CIFAR-100 作为起点；

视时间可扩展到 Tiny-ImageNet 或 ImageNet 子集。

9.2 实现步骤

使用预训练 CLIP 提取图像特征 f_i 与文本特征 t_y；

冻结 CLIP 主干，训练 Dataset Adapter，得到适配后特征 g_i；

基于 g_i 计算静态指标：SA_i / Div_i / DDS_i；

在原始图像上以 K 折交叉验证训练代理模型（ResNet-18），按折记录训练子集与验证子集（OOF）的 logits 轨迹；

基于训练轨迹计算 A/B/C/R，基于 OOF 验证轨迹计算 T（TransferGainScore）与 V（PersistentDifficultyScore），融合并归一化得到 u_i；

使用 Ridge 回归将 u_i 回归到 (SA_i, Div_i, DDS_i)，学习权重 w，得到 ẇ；

构造最终评分函数 Score(x)，对训练集与未见样本进行打分；

基于 Score 实现：

简单排序数据选择（不同选样比例）；

Selection Optimization（全局优化子集选择）；

将本方法与以下 baseline 对比：

使用全数据训练；

随机选择；

仅用单指标（如 SA 排序）；

经典方法（EL2N、MoSo 等）；

在不同噪声水平、不同数据规模、不同类别不平衡设置下评估：

分类准确率；

训练成本（epoch / 时间）；

选择子集的稳定性与可解释性；

学到的权重 ẇ 的差异及其合理性分析。

9.3 预期输出

一套完整的样本评分与数据选择算法（含代码）；

实验结果表证明在同等选样比例下优于或不劣于已有方法；

一个可用于后续研究和工程实践的“样本质量评分函数”框架。
